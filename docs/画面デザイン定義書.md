# 画面デザイン定義書

## 1. UI方針
*   **「引き算の美」**
    *   余白を贅沢に使い、ユーザーの愛車（Vehicle）の写真が主役になるようにする。
    *   UI要素は極力控えめに配置する。
*   **没入型ナビゲーション**
    *   背景は深く、境界線は曖昧にする。
    *   画面の切り替えはフェードなどで「流れるような体験」を作る。
*   **情報の階層化**
    *   重要なスコア（AI診断結果など）は大きく表示する。
    *   補足情報は小さく、繊細なフォントで配置する。

## 2. カラーパレット
映画のような重厚感を出すため、彩度を抑えた配色にする。

*   **ベース（背景）**: `#0D1117`（ディープミッドナイト）
    *   真っ黒ではなく、わずかに青みを感じる深い紺色で奥行きを出す。
*   **文字**: `#E6EDF3`（オフホワイト） / `#8B949E`（セカンダリグレー）
    *   眩しすぎない白を使い、暗い場所でも目に優しい配色にする。
*   **強調（CTA）**: `#238636`（グリーン） / Hover: `#2EA043` / Disabled: `#1E5C27`
    *   主要な実行ボタンはこのグリーン系を標準とする。
    *   保存、編集開始、送信、探索開始、アップロード、画像取得などの通常アクションに使用する。
*   **補助強調**: `#D4AF37`（アンティークゴールド）
    *   高級感と「真鍮」のようなメカニカルな印象を与える。
    *   スコア表示、補助的な強調、画面固有の演出要素に限定して使用する。
*   **危険/警告**: `#E5534B`（フェードレッド）
    *   異音検知や消耗品の劣化など、緊急性の高い通知に使用する。

## 3. 角丸・影・フォント
*   **角丸**: 標準 `8px`
    *   通常のボタン、入力、汎用アクションは `8px` を基本とする。
    *   丸型・アイコン専用・機能専用ボタンのみ個別定義を許容する。
*   **影**: `box-shadow: 0 10px 30px rgba(0, 0, 0, 0.5);`
    *   カードなどが浮き上がって見えるよう、深めのドロップシャドウで質感を演出する。
*   **フォント**
    *   英数字: `Inter` または `Roboto Mono`（メーターのような精密感）
    *   日本語: `Noto Sans JP`（Medium/Light）
    *   文字の間隔（レタースペーシング）を少し広めに設定して洗練さを出す。

## 4. コンポーネント基準（ビジュアル）
*   **ボタン**
    *   通常アクションボタンは `SettingPage` の「編集」ボタンを基準とする。
    *   標準レイアウトは `min-height: 50px` 相当、`padding: 0.8rem 1.2rem`、`border-radius: 8px`、`font-size: 1rem`、`font-weight: 600` とする。
    *   主ボタンは背景 `#238636`、文字 `#FFFFFF`、Hover `#2EA043`、Disabled `#1E5C27` を使用する。
    *   副ボタンは背景 `#21262D`、文字 `#C9D1D9`、枠線 `1px solid rgba(240, 246, 252, 0.1)`、Hover `#30363D` を使用する。
    *   危険ボタンは赤系背景と赤系ボーダーを使用し、削除や破壊的操作に限定する。
    *   `LoginPage` と `OnboardingPage` は例外として個別デザインを許容する。
    *   丸型・録音・シャッター・閉じるボタンなど、操作形状そのものに意味がある機能専用ボタンは個別定義を許容する。
    *   文字はすべて日本語とする。
*   **入力（Input）**
    *   下線のみ、または非常に細い枠線。
    *   フォーカス時にゴールドの光が漏れるようなエフェクト。
    *   アイコンは使用しない。
*   **カード**
    *   背景は少し明るいグレー（`#161B22`）。
    *   境界線はなし。
    *   写真は `2.35:1` のアスペクト比で固定する。
*   **モーダル**
    *   背景にブラー（背景ぼかし）を効かせる。
    *   コンテンツが浮き上がっているように見せる。
